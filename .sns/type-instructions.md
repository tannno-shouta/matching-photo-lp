# 投稿タイプ定義 — 福岡垢抜けマッチングフォト

各投稿は以下のいずれか 1 タイプに属する。`config.json` の `typeMix` で配分を制御。
**4タイプは AIDA ファネルに対応**：alpha=Attention/Interest, beta=Desire, gamma=Action入口, delta=Action。

## 全タイプ共通：三層構造を意識する

| 層 | 何を語るか | 比喩素材 | メリット辞書 |
|---|---|---|---|
| 表層・中層（7〜8割） | 写真・垢抜けの直接効果 | 家・服・店構え・名刺・料理人 | いいね↑/マッチ率↑/返信率↑/初回デート率↑ |
| 深層（2〜3割） | 関係性・自信・ライフスタイル | 美容室の帰り道・整っている人の姿勢 | 自信↑/関係継続↑/選択肢↑ |

## 全タイプ共通：文体ルール（厳守）

- **基本タメ口**：です・ます調は使わない
- **断言は体言止め**：「〜だ」「〜だよ」「〜だね」を抜く
- **短文を重ねるリズム**
- **NG**：「マジで」「〜じゃん」「俺も〜だった！」（痛いカジュアル）
- **OK**：「〜てる」「〜になる」「〜と思う」「〜という話」

## alpha

【純観察・比喩フック型】（AIDA: Attention + Interest）

**役割**: 比喩で読み手の注意を止め、「自分のことだ」と気づかせる。

**構造テンプレ**:
```
[1] 比喩フック（家・服・店構え・名刺・料理人など）
[2] 橋渡し（「マッチングアプリも、同じ。」「写真も、同じ。」）
[3] 気づき / 強い断言（タメ口・体言止め）
[4] 余韻（任意）
```

**ルール**:
- URL 一切禁止、ハッシュタグ禁止、サービス名禁止
- 客観メリット辞書から1〜2個まで滲ませる（並列羅列は beta の役割）
- 押し付け説教はNG。読み手に判断させる構造で

**例文（表層）**:
```
中身が同じ家でも、
外見が整った物件と、ボロボロの物件があったら、
前者を選ぶ。当然のように。

マッチングアプリも、同じ。
中身を見てもらえるのは、
入口を抜けた人だけ。

順番として、まず写真。
```

**例文（深層・自信/関係性）**:
```
美容室で髪を整えた帰り道、
姿勢が変わる感覚、わかると思う。

整っている人は、
選ばれる回数より、
選べる回数が多い。

垢抜けは、写真の中だけで終わらない。
```

## beta

【ペイン直撃 → ベネフィット羅列型】（AIDA: Desire）

**役割**: ペインを名指して鏡になり、写真・垢抜けに投資するメリットを並べて欲求を立ち上げる。

**構造テンプレ（PAS）**:
```
[1] Problem（ペイン名指し）：「いいねが0で止まってる」「中身を磨こうとする」
[2] Agitate：「それは順番が違う」「中身は入口を抜けた人にしか届かない」
[3] Solution（ベネフィット羅列）：
   ・いいね数↑
   ・マッチ率↑
   ・返信率↑
   ・初回デート率↑
[4] 締め（タメ口・体言止め）
```

**ルール**:
- サービス名・LP URL は出さない（読み手に想起させる手前で止める）
- 「素材」「垢抜け」「入口」など固有語彙を使ってよい（連発しない）
- URL 禁止、ハッシュタグ禁止
- 箇条書き（・/→）でメリットを整理する

**例文（表層）**:
```
いいねが0で止まってる人ほど、
中身を磨こうとする。

気持ちはわかる。
でも、それは順番が違う。

写真を変えると、こう変わる。
・いいね数↑
・マッチ率↑
・返信率↑

中身を見てもらえるのは、入口を抜けた人だけ。
```

**例文（深層・差別化）**:
```
カメラマンに頼んだのに、変わらなかった人が多い。

理由はシンプルで、
カメラマンが提供してるのは「撮影技術」だけ。
素材（服・髪・眉・表情）は、別問題。

整える順番は、撮る前から。
それで、写真の上限は変わる。
```

## gamma

【サービス紹介・bio URL 誘導】（AIDA: Action 入口）

**役割**: サービスの構造（垢抜け＋撮る）を「他と違う理由」として紹介し、bio または LP へ誘導。

**構造テンプレ**:
```
[1] 構造の違いを提示（撮るだけ vs 垢抜け+撮る）
[2] 比喩で補強（料理人と食材、ヘアサロンと骨格診断、など）
[3] LP / LINE 相談誘導（bio）
```

**ルール**:
- LP URL を本文に出してOK（最大1つ）
- 「→ bio」「詳細はプロフィールリンクから」など bio 誘導も可
- ハッシュタグ最大 1 個（使わなくてもよい）
- CTAは具体ベネフィット明示型に：「LINEで相談、無料」など

**例文**:
```
マッチング写真サービス、全部同じに見えるけど、構造が違う。

・撮るだけ → 素材は変わらない
・垢抜け＋撮る → 素材ごと整える

同じ料理人でも、食材が違えば結果が違う。
それと同じ話。

→ matching-photo.vercel.app
```

```
カメラマンに頼んでも変わらなかった人へ。

理由はだいたい、撮る前の「素材」が整ってないから。
服・髪・眉・表情。

そこから一緒に整える、というアプローチがある。

→ bio から相談できる。
```

## delta

【サービス直接訴求・週 1 まで】（AIDA: Action）

**役割**: ペイン提示 → ベネフィット羅列 → サービスの構造的差別化 → LP 誘導。

**構造テンプレ**:
```
[1] ペイン名指し（マッチング停滞 / 写真変えても変わらなかった）
[2] ベネフィット羅列（客観メリット辞書から3〜5個）
[3] 差別化軸（垢抜け＋撮る、福岡）
[4] LP URL or LINE 誘導
```

**ルール**:
- 景表法 NG 表現禁止（絶対 / 必ず / 保証 / 100% / 誰でも / 最強 / 最高）
- 数値主張は「〜という傾向がある」「〜と言われている」程度に抑える
- 断定調を避け、提案調で
- URL 1 つまで
- ハッシュタグ最大 1 個

**例文（表層訴求）**:
```
いいね0で止まってる人へ。

写真を変えると、こう変わる。
・いいね数↑
・マッチ率↑
・返信率↑

ただし、撮るだけのサービスだと素材は変わらない。
撮る前の "垢抜け" まで含めて整える、という選択肢。

→ matching-photo.vercel.app
```

**例文（深層訴求・自信/関係性）**:
```
マッチして会えても、続かない人が多い。

理由はだいたい、写真と実物のギャップ。
加工で盛った写真は、最初の一回しか効かない。

整えるのは加工じゃなく、本物の素材。
そこから写真を撮ると、関係も続きやすい。

福岡で、撮る前から整える、という選択肢。

→ matching-photo.vercel.app
```
